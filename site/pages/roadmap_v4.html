Ça Swingue 4 · Fusion tri-outils

Roadmap

PWA + Dashboards Nantes + Coaching golf-suivi → un seul produit 3 couches.

🧭 Vision

Trois outils actuels indépendants — Ça Swingue (PWA pendant la partie), Nantes_Vigneux (dashboards après partie), golf-suivi (coaching entre parties) — sont fusionnés en une app unique avec base de données commune.

📱 Live → 📊 Stats → 🎯 Coach + Caddy

📅 Phases

Phase 1 — Fondations
2-3 semaines
  • docs/10_data_model.md — schéma TypeScript unifié
  • Setup esbuild + TypeScript config minimal
  • Migration localStorage → IndexedDB (wrapper)
  • Routeur SPA simple (3 onglets Live/Stats/Coach)
  • Tests unitaires sur migrations v2 → v4
Phase 2 — Couche Live
1-2 semaines
  • Porter playing view depuis v2
  • Porter score saisie + format engine
  • Porter partage Firebase
  • Parité v2 validée
Phase 3 — Couche Stats
2-3 semaines
  • Conversion XLSX Nantes → JSON
  • Dashboard WHS (onglet 1)
  • Dashboard par trou (onglets 2 + 3)
  • Dashboard N-derniers (onglets 4 + 5)
  • Export PDF
Phase 4 — Couche Coach + Caddy
3-4 semaines
4a — Coach (entre parties · hérité de golf-suivi)
  • Parseur MD golf-suivi
  • UI conversationnelle
  • Skill planifier-séance
  • Skill analyser-parcours
  • Skill bilan-progression
  • Sync .md bidirectionnelle (File System Access)
4b — Caddy (pendant partie · hérité de v3)
  • Import des structures Caddy v3 (PlayerProfile, GameContext, CaddyRule)
🎯 Préférences de jeu enrichies
  • Identifie les clubs les plus joués avec réussite (stats Nantes)
  • Repère les distances où tu es le plus à l'aise (ex. 120m wedge, 150m fer 8)
  • Stocke ces préférences dans PlayerProfile.preferences
🗺️ Stratégie multi-parcours
  • Règles qui combinent hole.distance + préférences joueur
  • Plan de jeu (drive + lay-up + attaque) sur n'importe quel parcours
  • Fonctionne sans carnet rempli à la main, depuis les données accumulées
  • Enrichissement par stats Nantes (% GIR, trous faibles, etc.)
  • Briefing pré-coup avancé (profil + stats + forme + conditions)
  • Bilan mi-parcours (après 9 trous)
  • Coaching post-erreur (mental adaptatif)
  • Mémoire vive (observations passives → profil)
  • (optionnel) Appels Claude API pour cas limites
Fusion naturelle : Coach et Caddy partagent le même profil joueur, stocké en .md dans golf-suivi et en objet dans IndexedDB.
Phase 5 — Polish
1-2 semaines
  • PWA install + offline
  • Notifications push
  • Widget watch-sized
  • Doc utilisateur
⏱️ Total estimé : 9-14 semaines (2-3 mois)

🔧 Stack cible

TypeScript sans framework · esbuild pour le bundle · Chart.js stats · Marked markdown · Firebase sync · IndexedDB local

1 SPA PWA unifiée · 3 onglets Live / Stats / Coach · mobile-first, dashboards tablette/desktop
